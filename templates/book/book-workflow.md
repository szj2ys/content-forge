## 三、质量自检

### 3.1 结构检查清单

#### 结构完整性
- [ ] 每章有且只有一个核心问题要回答
- [ ] 每一节都有明确的论点，不是简单罗列
- [ ] 节与节之间有逻辑递进关系（非平铺）
- [ ] 开篇有引入（为什么要读这章），结尾有总结（读完收获了什么）

#### 内容充实度
- [ ] 每个论点至少有 2 种支撑材料（案例+数据、案例+类比等）
- [ ] 关键案例已确认可获取且有授权
- [ ] 数据来源可靠，非孤证
- [ ] 有至少 1 个让读者"意想不到"的内容

#### 读者视角
- [ ] 站在目标读者角度重新读了一遍大纲，逻辑通顺
- [ ] 没有假设读者知道本章才要教的内容
- [ ] 专业术语首次出现时有解释
- [ ] 抽象概念有具体化表达（类比、图示、案例）

#### 可执行性
- [ ] 每节字数估算合理，总字数在预算范围内
- [ ] 所需资料/案例/数据已列出获取计划
- [ ] 预计写作难点已有应对方案

---

### 3.2 内容检查清单

#### 事实准确性
- [ ] 无事实性错误（人名、地名、时间、事件等）
- [ ] 数据准确且来源可查证
- [ ] 技术内容在当前环境下可复现（技术书籍适用）
- [ ] 案例真实或已注明为虚构/化名处理
- [ ] 引用内容准确，无断章取义

#### 版权合规
- [ ] 无抄袭或未授权的大段引用
- [ ] 所有引用均已注明出处
- [ ] 图片/图表已获使用授权或为原创
- [ ] 涉及真人真事已获得当事人许可或做脱敏处理

#### 专业准确性
- [ ] 专业术语使用准确规范
- [ ] 理论引用正确，无曲解原意
- [ ] 方法论经得起推敲，无逻辑漏洞
- [ ] 代码/公式/数据已验证（如适用）

#### 出版规范
- [ ] 字数符合出版要求
- [ ] 章节编号、标题层级格式统一
- [ ] 参考文献格式规范
- [ ] 图表编号连续、说明文字完整
- [ ] 无错别字、无语病
