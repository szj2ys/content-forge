## 一、整体审核 Checklist

> 从课程整体视角审核，确保结构完整、定位清晰。

### 1.1 定位与目标

- [ ] 课程标题准确反映内容，无夸大或误导
- [ ] 课程简介清晰说明了"学完能做到什么"
- [ ] 目标学员画像明确，前置要求合理
- [ ] 课程难度始终保持在目标学员的"舒适区边缘"

审核意见：________________________________________________________________________

### 1.2 结构与编排

- [ ] 总讲数合理（建议 8-30 讲，实际 ________ 讲）
- [ ] 模块划分逻辑清晰，有明确的学习路径
- [ ] 难度梯度合理：从易到难，循序渐进
- [ ] 前后讲之间有自然衔接，不突兀
- [ ] 没有重复讲解相同内容的情况
- [ ] 没有遗漏关键知识点

审核意见：________________________________________________________________________

### 1.3 完整性

- [ ] 课程介绍/宣传页已完成
- [ ] 所有讲次内容已完成
- [ ] 课后作业/练习已配齐
- [ ] 参考资料/补充材料已整理
- [ ] 课程总结/结语已完成

## 三、红线检查 Checklist

> 以下任何一项不通过，课程**不得上线**，必须修改后重新审核。

### 3.1 事实与准确性

- [ ] 无明显的事实性错误
- [ ] 数据、案例有可靠来源，非编造
- [ ] 引用他人观点/成果时有注明出处
- [ ] 技术类内容已在最新版本环境下验证

发现的事实问题：________________________________________________________________________

### 3.2 版权与合规

- [ ] 无未经授权的他人内容（文字、图片、代码、视频片段）
- [ ] 引用内容符合合理使用规范
- [ ] 无涉及政治敏感、宗教、歧视等争议内容
- [ ] 无虚假承诺（如"保证月入过万""学完年薪百万"）

发现的合规问题：________________________________________________________________________

### 3.3 技术与制作

- [ ] 音频清晰，无明显杂音、爆音、中断
- [ ] 视频画面清晰，字幕（如有）无错误
- [ ] 代码/演示可正常运行，截图与当前版本一致
- [ ] 文件命名规范，资料可正常下载/打开

发现的技术问题：________________________________________________________________________

### 3.4 用户体验

- [ ] 课程播放顺序正确
- [ ] 课程详情页信息与实际内容一致
- [ ] 配套材料已正确关联到对应讲次
- [ ] 已在至少 1 种终端（手机/电脑/平板）上完整试看
