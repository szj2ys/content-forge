## 为什么需要内容品控？

大多数创作者凭感觉写作——选题靠灵感、结构靠经验、质量靠运气。结果就是产出不稳定：状态好时写得不错，状态差时不知道哪里出了问题。

**ContentForge 的思路很简单：像产品研发一样做内容。** 有流程、有标准、有检查清单。不是限制创造力，而是把「基本功」固化下来，让你把精力花在真正需要创造力的地方。

灵感来自[得到品控手册](https://www.dedao.cn/)——业内最系统的内容品控体系之一。我们将其简化为适合**个人创作者和小型内容团队**的轻量版本：保留核心方法论，去掉组织层面的复杂度。

## 核心流程：4 步闭环

```
选题 Discover ──→ 创作 Create ──→ 审核 Review ──→ 发布 Ship
      ↑                                               |
      └──────────────── 复盘反馈 ──────────────────────┘
```

### 1. 选题 Discover

不拍脑袋，用评分卡和调研模板系统评估候选题目。
好内容始于好选题——在这一步投入足够的时间，后续一切事半功倍。

### 2. 创作 Create

不从零开始，用大纲模板和写作规范结构化产出。
模板不是八股文，而是脚手架：帮你快速搭起骨架，然后专注打磨血肉。

### 3. 审核 Review

不凭感觉，用检查清单和同行评审确保质量。
自检清单帮你捕获 80% 的常见问题，同行评审覆盖剩下的盲区。

### 4. 发布 Ship

不留遗漏，用发布清单和复盘模板闭环。
发布不是终点，复盘才是——每次发布都是下一次选题的输入。

## 30 分钟快速开始

```bash
# 第 1 步：获取项目
git clone https://github.com/szj2ys/content-forge.git
cd content-forge
```

| 步骤 | 操作 | 耗时 |
|:----:|------|:----:|
| 1 | Fork 这个仓库（或 `git clone`） | 1 分钟 |
| 2 | 进入 `templates/article/` 选择文章模板 | 2 分钟 |
| 3 | 用 `01-topic-research.md` 评估你的 3 个候选选题 | 10 分钟 |
| 4 | 选定 1 个后，用 `02-outline.md` 搭建文章骨架 | 5 分钟 |
| 5 | 按 `03-draft-guide.md` 的写作规范完成初稿 | 不限 |
| 6 | 用 `checklists/review-checklist.md` 逐项自检 | 5 分钟 |
| 7 | 发布，然后用 `checklists/ship-checklist.md` 复盘 | 5 分钟 |

> **提示：** 第一次使用建议从一篇短文章开始，走完完整流程后再应用到大型项目。

## 方法论来源

本项目的方法论融合了以下实践：

- **得到品控手册** — 选题评估、品控标准、交付规范的核心框架
- **内容营销最佳实践** — 受众调研、内容分发、效果度量的行业经验
- **敏捷开发思想** — 迭代交付、MVP 思维、快速反馈的工程方法

我们不生搬硬套任何一个体系，而是取其精华，适配个人创作者的真实工作场景。
