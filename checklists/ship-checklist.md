# 发布阶段检查清单

> 写完不等于完成，发布才是起点。发布阶段的目标是：让好内容以最佳状态到达读者，并在发布后持续观察和学习。

---

## 发布前准备

### 内容包装

- [ ] **标题定稿**：终版标题是否简洁、准确、有吸引力？（≤25 个字为佳）
- [ ] **副标题/摘要**：是否用 1-2 句话概括核心价值？（读者据此决定是否点开）
- [ ] **配图就绪**：封面图/题图是否清晰、与主题相关、尺寸符合平台要求？
- [ ] **正文配图**：文中图片是否全部加载正常、有替代文字（alt text）？
- [ ] **作者信息**：署名、头像、个人简介是否准确且最新？

### 排版与格式

- [ ] **排版检查**：标题层级、段落间距、字体大小是否统一规范？
- [ ] **代码格式**：代码块是否有语法高亮？代码是否可以正确复制？
- [ ] **列表与引用**：格式是否正确渲染？缩进是否一致？
- [ ] **移动端预览**：在手机上完整浏览一遍，确认阅读体验良好
- [ ] **暗色模式检查**：如果平台支持，确认暗色模式下内容可正常阅读

### 链接与引用

- [ ] **内链检查**：文中所有链接均可正常跳转
- [ ] **外链检查**：引用的外部链接均有效且指向正确内容
- [ ] **参考资料**：如有参考文献列表，格式统一且完整
- [ ] **CTA（行动号召）**：文末的下一步引导是否清晰？链接是否正确？

### 发布策略

- [ ] **发布时间确定**：选择目标读者活跃的时段（记录选定时间：___）
- [ ] **发布渠道确认**：主发布渠道 + 分发渠道列表已明确
- [ ] **各渠道格式适配**：不同平台的格式要求是否分别满足？
- [ ] **SEO 基础**：关键词、描述、URL slug 是否已优化？（如适用）
- [ ] **分享预览**：在微信/Twitter/社交平台分享时的卡片预览是否正常？

---

## 发布执行

- [ ] 在主渠道发布内容
- [ ] 确认发布后页面正常加载，格式无异常
- [ ] 在各分发渠道同步发布（根据渠道特点调整标题和摘要）
- [ ] 在相关社群/朋友圈进行首轮推广（附简短推荐语，不要只丢链接）
- [ ] 记录发布时间和各渠道链接

---

## 发布后 72 小时跟踪

### 第 1 小时

- [ ] 检查发布页面是否正常（无格式错误、图片丢失、链接失效）
- [ ] 回复首批评论和反馈（前几条互动会影响后续读者参与度）
- [ ] 如发现错误，立即修正并记录（注明"已更新"）

### 第 24 小时

- [ ] 查看核心数据：阅读量、阅读完成率、互动数（评论/点赞/分享）
- [ ] 数据是否符合预期？如有明显偏差，分析可能原因
- [ ] 处理所有新增评论和反馈
- [ ] 是否需要追加推广？（如数据良好但曝光不足）

### 第 72 小时

- [ ] 汇总最终数据快照：阅读量___、完成率___、互动数___、分享数___
- [ ] 整理有价值的读者反馈（尤其是你没预料到的问题和视角）
- [ ] 是否有后续内容的机会？（读者追问最多的方向）
- [ ] 数据录入内容库，便于后续复盘对比

---

### 回顾四问

- [ ] **做对了什么？** 哪些环节的产出超出预期？为什么？
- [ ] **做错了什么？** 哪些环节出了问题？根因是什么？
- [ ] **意外发现了什么？** 有哪些计划外的收获或教训？
- [ ] **下次改进什么？** 列出 1-3 条具体的、可执行的改进项

### 流程效率回顾

- [ ] 记录各阶段实际耗时：选题___h → 创作___h → 审核___h → 发布___h → 总计___h
- [ ] 与预估时间对比，偏差是否可接受？
- [ ] 哪个环节效率最高？哪个环节耗时最不值得？
- [ ] 流程中哪些工具/模板最有用？哪些可以简化或去掉？

---

**通过标准**：发布后 72 小时跟踪完成，复盘表格填写完毕。

**常见失败模式**：
- 发布后就"翻篇了"，不跟踪数据（失去了最重要的学习机会）
- 只看阅读量不看完成率（高阅读低完成 = 标题好但内容没留住人）
- 复盘流于形式，写"下次注意"而非具体可执行的改进
- 不记录耗时，无法评估流程效率是否在持续提升
