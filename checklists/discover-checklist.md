# 选题阶段检查清单

> 好内容始于好选题。跳过选题直接写作，就像不看地图就上路——可能到达某处，但大概率不是你想去的地方。

---

## 调研完整性

- [ ] 收集 ≥ 5 个候选选题，记录每个选题的来源和初步判断
- [ ] 对每个候选选题使用评分卡打分（需求强度 / 独特价值 / 能力匹配 / 竞品空间 / 传播潜力，各 1-5 分）
- [ ] 完成竞品分析：找到排名前 3 的同类内容，记录其优势和不足
- [ ] 确定差异化角度：明确"别人没说过"或"别人没说好"的切入点
- [ ] 汇总调研结论，形成候选选题排序表

## 用户需求验证

- [ ] 目标读者画像明确：能用一句话描述"这篇内容写给谁"
- [ ] 核心痛点已确认：读者在这个问题上遇到的具体困难是什么
- [ ] 需求真实存在：有搜索数据、社区讨论、身边反馈等至少一种证据支撑
- [ ] 需求紧迫程度评估：这是"想了解"还是"急需解决"
- [ ] 读完之后的预期收获明确：读者能带走什么具体可执行的东西

## 可行性确认

- [ ] 能力匹配：自己在该主题上有足够的经验或认知深度
- [ ] 素材充足：已有或能获取到足够的案例、数据、引用来支撑论点
- [ ] 时间合理：预估创作周期在可接受范围内（标注预估时间：___小时）
- [ ] 没有硬伤：不涉及自己不确定的事实判断或敏感领域
- [ ] 时效性检查：如果与时事相关，发布时是否仍然有价值

## 最终决策

- [ ] 从候选中选定 1 个选题（评分最高 + 直觉确认）
- [ ] 写出一句话定位：「本文帮助 [目标读者] 解决 [核心问题]，通过 [独特角度]」
- [ ] 定义成功标准：发布后用什么指标判断这篇内容是否成功
- [ ] 淘汰选题归档：记录淘汰原因，留待未来重新评估

---

**通过标准**：以上所有必填项均已勾选，且最终决策部分完整填写。

**常见失败模式**：
- 只收集了 1-2 个候选就匆忙决定（选择太少 = 没有选择）
- 评分卡全凭感觉打分，没有竞品分析支撑
- 差异化角度模糊，说不清"为什么读者要看我写的而不是别人的"
- 跳过需求验证，写自己想写的而非读者需要的
