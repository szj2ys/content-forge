# 创作阶段检查清单

> 创作不是一气呵成的灵感迸发，而是"大纲→初稿→四遍修改"的工程化过程。每一步都有明确的完成标准。

---

## 大纲阶段

- [ ] 核心论点明确：能用一句话说清楚"这篇文章要证明什么"
- [ ] 结构类型已选定（递进式 / 并列式 / 问题-方案式 / 故事线 / 对比式）
- [ ] 主要段落规划完成：每个段落有标题 + 一句话概括 + 支撑素材
- [ ] 逻辑链条通顺：从 A 到 B 到 C，每一步推导读者都能跟上
- [ ] 开头策略确定：用什么钩子抓住读者（场景 / 数据 / 问题 / 反直觉观点）
- [ ] 结尾策略确定：给读者什么收尾感（行动号召 / 思考延伸 / 金句总结）
- [ ] 篇幅预估合理：与主题深度匹配，不膨胀也不单薄
- [ ] 大纲已通读一遍，确认"如果按这个写，成品不会跑偏"

## 初稿阶段

- [ ] 按大纲逐段完成初稿，不回头修改（先完成，再完美）
- [ ] 每个核心观点都有支撑：案例、数据、类比、引用——至少用一种
- [ ] 关键段落的过渡自然：读者不会感到突兀的跳跃
- [ ] 初稿已写完，通读一遍确认"骨架完整，没有缺失的重要段落"
- [ ] 标记需要补充的地方（用 `[TODO]` 标注），不因素材不足而卡住
- [ ] 放置至少 2 小时再进入修改阶段（让大脑重置，才能看到问题）

## 修改阶段

修改分四遍进行，每遍只关注一个维度。**不要在一遍里试图解决所有问题。**

### 第一遍：结构修改

- [ ] 整体结构是否支撑核心论点？有没有跑题的段落？
- [ ] 段落顺序是否最优？调换任意两段是否会更好？
- [ ] 每个段落的长度是否合理？是否有段落需要拆分或合并？
- [ ] 开头是否在前 3 句话内抓住读者？
- [ ] 结尾是否有力？读者读完最后一段会有什么感受？
- [ ] 删除所有不为核心论点服务的内容（即使写得很好）

### 第二遍：内容修改

- [ ] 每个事实性陈述是否准确？来源是否可靠？
- [ ] 案例和数据是否充分支撑论点？有没有"举例不当"的情况？
- [ ] 是否存在逻辑漏洞？读者可能提出什么反驳？是否已预先回应？
- [ ] `[TODO]` 标记是否全部处理完毕？
- [ ] 是否有重复表达的内容？合并或删除
- [ ] 专业术语是否都有解释或上下文推断？

### 第三遍：表达修改

- [ ] 句子是否简洁？删除所有"其实""基本上""某种程度上"等弱化词
- [ ] 是否存在长难句？超过 40 字的句子考虑拆分
- [ ] 用词是否精准？避免模糊的"很多""经常""一些"
- [ ] 语气是否一致？全文的人称、口吻、正式程度是否统一
- [ ] 类比和比喻是否贴切？会不会造成误解？
- [ ] 标题和小标题是否吸引人且准确？

### 第四遍：体验修改

- [ ] 在手机上通读一遍，排版是否友好？
- [ ] 段落之间是否有足够的呼吸感？（连续密集文字不超过 5 行）
- [ ] 重点内容是否有视觉突出？（加粗、列表、引用块等）
- [ ] 图片/图表是否清晰且有说明？
- [ ] 全文阅读时间是否合理？（可标注预计阅读时间）
- [ ] 以"第一次读这篇文章的人"的视角完整阅读一遍，记录卡顿的地方

---

**通过标准**：四遍修改全部完成，所有勾选项确认，`[TODO]` 标记归零。

**常见失败模式**：
- 初稿没写完就开始修改（反复打磨第一段，后面草草收场）
- 四遍修改混在一遍做（结果每个维度都没改透）
- 舍不得删掉"写得好但跑题的段落"
- 跳过手机端体验检查（大量读者在手机上阅读）
